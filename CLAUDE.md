# honeypot-trap / decoyops

## Workflow

- This is a learning project: teach concepts before writing code. Do not write
  code until the user explicitly asks for it in that step.
- GitHub repo: https://github.com/Pranavc07/honeypot-trap (public).
- At the end of each session with meaningful changes, commit and push to
  `origin/master` automatically — no need to ask for confirmation each time.
  Just notify the user what was pushed (files changed + one-line summary).
  Do not push secrets, `.sqlite3` DB files, or `.claude/settings.local.json`
  (already in `.gitignore`).
